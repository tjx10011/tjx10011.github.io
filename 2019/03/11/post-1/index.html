9102年的初心者科学上网指南（二）

发表于 2019-03-11 | 更新于 2019-03-14 | 分类于 技术 ， 网络

VPS订购后步骤

选择系统

订购VPS时一般会让你选择系统。这里推荐ubuntu，因为centos的部分命令不同有些不不习惯。版本个人选择较新的版本(一般为18.04)。

配置防火墙(GCP，Azure等适用)

购买VPS后这些商家可能需要手动更改防火墙规则以允许流量传输。

  • GCP:创建VM实例时勾选允许HTTP/HTTPS流量。然后，在左侧菜单中选择VPC网络，设置防火墙规则。添加规则，内容为0.0.0.0/0，优先级填写1，来允许所有流量。然后选择菜单中的IP地址，将其中IP地址的“临时”属性改为“静态”。这个就是你VPS的IP。
  • Azure需要在”安全组”中进行类似设置。

配置NAT

NAT类型的VPS需要配置端口映射。商家会提供控制面板的访问方式，在控制面板中会有NAT一项可以设置。一般可以看到你的公网IP，已经有一个默认创建的“内网22”映射到“公网XXX”。可以添加映射路径。记住你创建的映射。在配置v2ray时，服务端配置文件端口填写内网数值，客户端配置文件填写公网IP，端口填写公网数值以正确对接。

其它控制面板操作

在商家提供的网页控制面板，可以进行重装，重启，开关，监控系统状况等操作，比较有用。

连接VPS

GCP网页控制台“实例列表”点击SSH即可在网页端连接。
一般需要用SSH客户端进行连接。这里推荐MobaXterm，因为个人在用，比较方便(wsl也可以用它开gui，具体另外写文章)，颜值也可以。下载安装后，点击上方一行选项中最左边的Session，选择最左边的SSH，输入VPS的公网IP，端口(一般为22，或者是NAT对应的公网端口)，用户名(一般为root)等信息，确认然后连接，输入密码(一般商家提供或者自行在VPS控制面板修改)，就可以连接VPS开始操作。

配置前准备

基础操作

  • 如果登录用户不是Root，输入sudo su切换为root用户。
  • 执行apt-get update更新软件包目录。
  • 如果之后的安装过程提示有某个软件包没有安装，执行apt install xxx进行安装。
  • 常用命令，如cd /xxx/xxx/跳转到某个路径，ls显示当前目录下的文件，想查找某一命令用法搜索引擎即可。
  • nano文本编辑器。比较简单，容易上手。nano即可打开。nano filename可以打开指定文件。

跑分(可选)

个人使用Zbench，大佬制作，功能较为全面。使用方法：
wget -N --no-check-certificate https://raw.githubusercontent.com/FunctionClub/ZBench/master/ZBench-CN.sh && bash ZBench-CN.sh
之后按照提示操作即可。可以用来测试VPS性能，以及装逼。

bbr的安装

Bbr可以有效优化tcp传输，反映在科学上网的速度更快。
部分商家可能已经优化，不需要或者禁止安装bbr，请注意这一点。

OpenVZ安装

采用南琴浪大佬的脚本。 但是由于部分OpenVZ机器是由其他虚拟化机器(母鸡)开出来的虚拟机(小鸡)，所以直接安装会报错。以下介绍解决方法：
wget https://github.com/tcp-nanqinlang/lkl-rinetd/releases/download/1.1.0/tcp_nanqinlang-rinetd-debianorubuntu.sh
以上命令下载脚本。
nano tcp_nanqinlang-rinetd-debianorubuntu.sh
打开编辑器编辑脚本文件。
在25行check_ovz处四行每行前添加#(到}处为止)。
再到文件88行左右，在check_ovz行首添加#。 按Ctrl+X选择保存并退出。
以上操作，把脚本中检查虚拟化技术的部分注释掉，避免报错。前提是你清楚自己的VPS是OpenVZ虚拟化的。
然后执行bash tcp_nanqinlang-rinetd-debianorubuntu.sh，按照屏幕提示操作即可。
